## 什么是 Cowork？

**第1课：** 告诉 Claude *"阅读 START-HERE.md 并开始第1课"*

**Claude Cowork 是面向非开发者的 Claude Code。** 同样强大的 AI 代理，封装在不需要终端的界面中。

你给 Claude 访问电脑上某个文件夹的权限。Claude 然后可以读取、编辑和创建该文件夹中的文件——整理下载内容、从收据构建电子表格、从零散笔记起草报告等。

**与普通 Claude 的关键区别：** Cowork 生成实际的文件，而不是你需要复制粘贴的文本。要求一个费用电子表格，得到一个带有可用公式的真实 `.xlsx` 文件。要求一个演示文稿，得到一个带有幻灯片的真实 `.pptx` 文件。

### 比喻

| 工具 | 角色 | 功能 |
|---|---|---|
| **Claude** | 研究员 | 回答问题，进行研究 |
| **Claude Code** | CTO | 编写和编辑代码 |
| **Claude Cowork** | COO | 运营你的工作生活——文件、文档、研究 |

Anthropic 用大约10天时间构建了 Cowork。使用 Claude Code。

### 开始使用

2. 在侧边栏点击 **"Cowork"**

4. 开始委托任务

**重要：** 从一个专门的工作文件夹开始。不要在第一天就把 Claude 指向你整个文档文件夹。

## 思维转变

使用 Cowork 时最大的错误是把它当作带文件夹访问权限的 ChatGPT。

**错误方法：**

- 提问 → 得到答案 → 复制粘贴到文档

**正确方法：**

- 描述结果 → 离开 → 回来看到完成的工作

正如 Anthropic 所说：*"感觉不太像来回对话，更像是给同事留言。"*

如果你继续把它当作聊天界面，你只能获得10%的价值。

## "完成"框架

**第3课：** 告诉 Claude *"阅读 START-HERE.md 并开始第3课"*

输出质量与输入质量成正比。

在委托任何任务之前，回答三个问题：

### 1. "完成"是什么样子？

如果你无法想象完成状态，Claude 也无法做到。

| ❌ 模糊 | ✅ 具体 |
|---|---|
| "创建一个费用报告" | "创建一个 Excel 电子表格，包含日期、供应商、金额和类别列，按日期排序，底部有合计总额" |

### 2. Claude 需要什么上下文？

- 你的命名规范？

- 你的文件夹结构偏好？

- 工作文档和个人文档的区别？

如果你不告诉 Claude，它会做假设。有时是对的。有时不是。

### 3. 有哪些约束条件？

- "不要删除任何文件，只移动它们。"

- "保留原始文件名但添加日期前缀。"

- "只处理最近90天的文件。"

Claude 无法读懂你未说明的限制。

**最安全的默认设置：** 除非你特别想要删除，否则总是说"不要删除任何东西"。

## 文件夹设置

**第5课：** 告诉 Claude *"阅读 START-HERE.md 并开始第5课"*

不要给 Claude 访问随机文件夹的权限。创建一个系统。

### 推荐结构

```
Claude-Work/
├── inbox/ ← 把文件放在这里让 Claude 处理
├── processed/ ← Claude 把完成的工作移到这里
├── outputs/ ← Claude 在这里创建新文件
└── reference/ ← Claude 应该读取但不修改的文件
```

### 告诉 Claude 关于它的信息

在你的第一个提示中包含这个：

```
inbox 文件夹包含要处理的文件。完成后将它们移到 processed。
在 outputs 中创建任何新文件。reference 文件夹是只读的——不要修改那些文件。
```

这创建了一个可重复的工作流程。把文件放入 inbox，告诉 Claude 处理，在 outputs 中找到结果。

## 文件操作

**第2和第4课：** 告诉 Claude *"阅读 START-HERE.md 并开始第2课"*

这是 Cowork 的超能力：**内容感知的文件整理**。

### 有什么不同

普通的文件整理看文件名。`IMG_4521.png` 放入"Images"。

Cowork 看**内容**。那个 `IMG_4521.png`？Claude 可以看到它实际上是咖啡供应商的收据。它放入"Receipts/Coffee-Suppliers"。

### 规模

人们已经在几分钟内整理了300多个文件。多年混乱的下载文件夹——通过实际阅读每个文件的内容进行分类。

### 模式

1. 把 Claude 指向一个混乱的文件夹

2. 用完成框架给出清晰的委托

3. 请求一个变更日志以便你可以审查决策

4. 让它运行

### 示例委托

```
根据内容将这个文件夹中的所有文件整理到逻辑类别中。
在可识别日期的地方使用 YYYY-MM-DD 前缀重命名文件。
创建合理的子文件夹。在 ORGANIZATION-LOG.md 中记录每个决策，
显示每个文件的前后对比。不要删除任何东西。
```

## 研究与综合

**第6课：** 告诉 Claude *"阅读 START-HERE.md 并开始第6课"*

这是 Cowork 节省时间的地方。

### 问题

你有20份文档——客户访谈、调查结果、支持工单。你需要模式和洞察。

手动方式：阅读每份文档，做笔记，交叉引用，撰写发现。数小时的工作。

### Cowork 方法

Claude 同时阅读所有文档并进行综合——不仅仅是总结每一份，而是在它们之间建立联系。

客户 A 说了一件事。客户 B 说了相关的事。客户 C 与两者矛盾。Claude 捕捉所有这些并引用来源。

### 请求什么

```
阅读这个文件夹中的所有文档。创建一个综合报告，包含：
- 执行摘要
- 关键主题（附支持性引用和源文件引用）
- 来源之间的矛盾
- 按 [细分/类别/来源类型] 分组的模式
- 提到的具体 [请求/建议/洞察]
- 仍未回答的问题
```

### 这适用于什么

- 客户反馈综合

- 竞争研究

- 文献综述

- 多次会议的会议笔记

- 访谈记录

- 任何多来源分析

## 文档创建

**第8课：** 告诉 Claude *"阅读 START-HERE.md 并开始第8课"*

Cowork 生成**实际的文件**，而不是你需要复制粘贴的文本。

### 每种格式的功能

| 格式 | 能力 |
|---|---|
| **Excel (.xlsx)** | 可用公式（SUM、VLOOKUP、IF）、条件格式、多工作表、图表、数据验证 |
| **PowerPoint (.pptx)** | 带布局的真实幻灯片、演讲者备注、逻辑流程——不只是项目符号 |
| **Word (.docx)** | 正确的标题、目录、格式化表格 |
| **PDF** | 表单填写、文档组装 |

### 示例：收据 → 费用追踪器

```
这个文件夹中有收据照片。创建 expense-tracker.xlsx，包含：
- 数据工作表：日期、供应商、类别、金额、支付方式
- 按类别合计的公式
- 超过$100的费用的条件格式
- 带图表的摘要工作表
- 不清楚收据的标记项工作表
```

你得到一个真实的 Excel 文件。打开它，编辑它，分享它。

## 子代理

**第7课：** 告诉 Claude *"阅读 START-HERE.md 并开始第7课"*

当任务有独立部分时，Claude 会启动多个工作器同时处理它们。

### 工作原理

不是按顺序处理10个文件（50分钟），Claude 并行处理所有10个（5分钟），然后综合结果。

### 何时明确请求它们

| 模式 | 示例提示 |
|---|---|
| **供应商分析** | "启动子代理独立研究这四个供应商。然后综合成对比报告。" |
| **多视角** | "使用三个子代理分析这个决策：财务影响、客户体验、运营风险。综合发现。" |
| **大型文档集** | "使用子代理并行处理这50份记录。提取主题，然后综合成一份报告。" |

### 每个子代理拥有的

| ✅ 有 | ❌ 没有 |
|---|---|
| 完整的文件系统访问 | 与其他代理的通信 |
| 所有 Claude 能力 | 处理期间的共享状态 |
| 独立的上下文 | 其他代理工作的知识（直到综合） |

## 浏览器自动化

**第9课：** 告诉 Claude *"阅读 START-HERE.md 并开始第9课"*

通过 **Claude in Chrome** 扩展，Cowork 可以控制你的浏览器。

### 效果好的用途

| 用例 | 示例 |
|---|---|
| **竞争研究** | 访问定价页面，提取层级和功能，创建对比电子表格 |
| **数据提取** | 从你已登录的仪表板提取数据 |
| **公开数据收集** | 查找 SEC 文件，提取财务数据，创建对比表 |

### 实话实说

浏览器自动化**时好时坏**。页面加载奇怪。元素不响应。网站阻止自动化。

有时效果很好。有时会卡住。如果 Claude 卡住了，告诉它重试或跳过那部分。

### 安全警告

浏览器访问带来提示注入风险。恶意网站可能试图操纵 Claude 的行为。

**规则：**

- 坚持使用可信网站

- 不要把 Claude 发送到随机 URL

- Claude 会在提交表单或采取不可逆操作前询问你

## 技能系统

**第5和第11课：** 告诉 Claude *"阅读 START-HERE.md 并开始第5课"*

技能是 Claude 遵循的可重用指令集。

### 技能的作用

不用每次会话都重新解释你的偏好，创建一次技能并永久引用它。

### 创建自定义技能

在你的工作文件夹中放入一个 markdown 文件：

```
# 我的报告风格指南

## 格式
- 总是使用项目符号，不用段落
- 顶部包含执行摘要
- 以"下一步"部分结束

## 语调
- 专业但不生硬
- 没有解释不使用术语
- 主动语态

## 状态报告结构
1. 摘要（最多3个项目符号）
2. 本周完成
3. 进行中
4. 阻碍
5. 下周
```

然后告诉 Claude：*"阅读 style-guide.md 并在你创建的所有文档中遵循它。"*

### 内置技能

Cowork 附带 Excel、PowerPoint、Word 和 PDF 创建的技能。当你请求这些文件类型时，它们会自动加载。

## AI 员工模式

**第10课：** 告诉 Claude *"阅读 START-HERE.md 并开始第10课"*

这是解锁 Cowork 全部价值的思维方式。

### 旧方式（聊天机器人思维）

```
给任务 → 等待 → 得到结果 → 给下一个任务 → 等待 → 得到结果
```

### AI 员工方式

```
批量倾倒所有任务 → 移交它们 → 做别的事 → 回来看到完成的工作
```

### 批量策略

不是三个单独的会话：

```
"整理我的收据" → 等待 → 完成
"创建费用报告" → 等待 → 完成
"为我的会计师总结" → 等待 → 完成
```

这样做——一个会话：

```
"我有收据照片在 /receipts。整理它们，创建一个费用追踪电子表格，
按类别汇总，并为我的会计师准备一份摘要文档。不要删除任何原始文件。"
```

Claude 处理整个工作流程。你去做别的事情。回来时一切都完成了。

## Cowork vs Claude Code

| 方面 | Claude Code | Claude Cowork |
|---|---|---|
| **目标用户** | 开发者 | 非开发者 |
| **界面** | 终端 | 桌面应用 |
| **主要用例** | 编码、Git、部署 | 文件、文档、研究 |
| **技术要求** | 命令行熟练度 | 无 |
| **文件操作** | 代码文件 | 所有文件类型 |
| **最适合** | 软件开发 | 通用生产力 |

**关键洞察：** 它们使用相同的技术。只是不同的界面。

## 局限性

### 效果不好的

| 局限 | 原因 |
|---|---|
| **实时协作** | 不支持多用户同时编辑 |
| **大型二进制文件** | 视频编辑、大型图像处理 |
| **需要登录的网站** | 无法处理复杂认证流程 |
| **精确设计工作** | 无法像素级精确放置元素 |
| **长时间运行的任务** | 会话超时限制 |

### 当前边界

- **平台**：仅 macOS（Windows 即将推出）

- **文件大小**：大文件可能很慢

- **并发**：一次一个活动会话

- **撤销**：没有内置撤销——依赖 Git 或备份

- [ ] 检查文件变更

- [ ] 报告问题
