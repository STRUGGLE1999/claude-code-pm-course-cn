# 2.3: 产品策略

- **完成时间：** 60-90 分钟

- **前置要求：** 模块 1.4（任务编排），模块 1.5（自定义 Sub-Agents）

**在 Claude Code 中开始本模块：** 运行 `/start-2-3` 启动交互式体验。

## 概述

模块 2.3 教你如何将 AI 作为思考伙伴来制定产品策略。你将使用 Rumelt 的策略内核框架（诊断 → 指导方针 → 一致行动）来构建战略思维，用魔鬼代言人压力测试选择，并将策略文档转化为高管演示文稿。

**核心要点：** AI 不能替你做战略决策，但可以帮助你更快地研究、严谨地应用框架、挑战假设、清晰沟通。

## 什么是产品策略？

产品策略是关于**在哪里集中资源以及如何创造竞争优势**的一系列艰难选择。

### 策略 vs 非策略

| 非策略 | 真正的策略 |
|---|---|
| "收入增长 50%"（目标） | "专注于面向 SMB 的语音优先 AI，明确不在广度上与 Notion 竞争"（清晰选择 + 权衡取舍） |
| "构建 AI 聊天、语音、自动化"（功能） | "在有限资源下深入发展我们能独特获胜的一项能力"（聚焦方向） |
| "成为最好的生产力工具"（愿景） | "初期补贴 AI 成本以通过使用数据和留存建立防御性"（定位） |

### AI 在策略中的角色

| AI 不能做 | AI 能做 |
|---|---|
| 做权衡取舍（需要判断力） | 并行研究竞争对手 |
| 提供关于你市场的深层上下文 | 严谨地应用框架 |
| 承诺做出艰难选择 | 挑战假设（魔鬼代言人） |
|  | 综合文档并创建演示文稿 |

## Rumelt 的策略内核框架

### 三部分结构

```
┌─────────────────────────────────────────┐
│ 1. 诊断 (DIAGNOSIS)                      │
│ 挑战/机会是什么？                         │
└─────────────────────────────────────────┘
 ↓
┌─────────────────────────────────────────┐
│ 2. 指导方针 (GUIDING POLICY)             │
│ 我们的总体方法是什么？                     │
│（在哪里竞争，如何获胜）                    │
└─────────────────────────────────────────┘
 ↓
┌─────────────────────────────────────────┐
│ 3. 一致行动 (COHERENT ACTIONS)           │
│ 我们将采取哪些具体步骤？                   │
│（相互强化并与方针一致的举措）              │
└─────────────────────────────────────────┘
```

### 1. 诊断

基于真实数据，对战略挑战或机会的清晰、可执行的陈述。

**好的诊断：**

AI 领域正在快速升温。Notion、Linear 和 Asana 在过去 6 个月都推出了 AI 功能。作为一个拥有 2 人 AI 团队和有限预算（约 $50k/季度）的小玩家，我们无法在广度上竞争。我们需要找到一个能在有限资源下独特获胜的防御性位置。

**差的诊断：**

竞争对手都在做 AI（太模糊，没有洞察）

### 2. 指导方针

你的总体方法，对**在哪里竞争和如何获胜做出明确的权衡取舍**。

**好的指导方针：**

我们将专注于面向 SMB（5-20 人团队）的语音优先 AI，明确不在广度上与大型玩家竞争。我们将通过小团队需要但专注企业的竞争对手不会构建的 AI 功能来差异化。我们将在基础定价中补贴 AI 成本以推动采用，接受初期较低利润率，通过使用数据和留存建立防御性。

**差的指导方针：**

我们将构建出色的 AI 功能并在市场上获胜（没有选择，没有权衡）

### 3. 一致行动

实施指导方针并相互强化的具体、协调的举措。

**好的一致行动：**

```
Q1 2026:
- 扩展语音到会议笔记 + 任务分解（深化语音能力）
- 推出 SMB 专用模板（5-20 人团队工作流）
- A/B 测试 AI 在基础价格 vs 高级层级

Q2 2026:
- 语音协作功能（多用户语音会话）
- SMB 管理仪表板（小团队负责人的使用追踪）
- 根据 Q1 数据优化定价
```

## 策略开发工作流程

```
1. 竞品研究
 ↓（使用并行 agents 和 WebSearch）
2. 战略选择
 ↓（做出 5 个关键权衡决策）
3. 魔鬼代言人
 ↓（压力测试每个选择）
4. 综合
 ↓（AI 使用 Rumelt 内核创建策略文档）
5. 演示文稿
 ↓（使用 pptx skill 制作高管幻灯片）
```

### 1. 使用并行 Agents 进行竞品研究

同时启动多个 agents 研究不同竞争对手。

```
Spin up 3 agents to research the competitive AI landscape:
- Agent 1: Research Notion's AI features, pricing, and positioning
- Agent 2: Research Linear's AI features, pricing, and positioning
- Agent 3: Research Asana's AI features, pricing, and positioning

For each competitor, I need:
- What AI features they've launched
- How they position and price AI
- What seems to be working vs. not working
- Where there might be whitespace for us
```

**为什么并行 agents 有效：** 比顺序研究快 3 倍，每个 agent 使用 WebSearch 获取最新信息。

### 2. 做出战略选择

回答 5 个强制真正权衡的多选题：

| 选择 | 选项 |
|---|---|
| **#1: 聚焦 vs 广度** | A) 深入发展一项能力
B) 将 AI 扩展到所有功能
C) 合作获取能力 |
| **#2: 竞争应对** | A) 竞相超越创新
B) 差异化竞争
C) 忽略并专注路线图 |
| **#3: 商业模式** | A) 高级 AI 层级（利润优先于采用）
B) 补贴以推动采用
C) 基于使用量定价 |
| **#4: 产品范围** | A) AI 作为产品
B) AI 作为增强
C) AI 用于特定任务 |
| **#5: 风险承受** | A) 快速行动，承担风险
B) 稳健可防御
C) 观望学习 |

你的 5 个答案构成你的指导方针。

### 3. 魔鬼代言人 - 压力测试选择

在每个选择之后，用 AI 挑战你的思考：

```
用户选择：A) 深入发展语音

魔鬼代言人回应：
你把一切都押在语音上。但如果语音在 6 个月内变成商品化呢？
OpenAI 刚刚以 1/10 的成本发布了更好的语音识别。如果每个人都有语音，
你就把策略建立在不再差异化的东西上了。只有 2 人的团队，
你没有资源来转型。这不是很冒险吗？

用户然后：坚持选择 或 重新考虑
```

**为什么这有效：** 在承诺之前强迫你思考不利因素。从 AI 听到尖锐问题总比从 CEO 那里听到好。

### 4. 综合策略文档

一旦你做出 5 个战略选择，AI 使用 Rumelt 内核将它们综合成完整的策略文档：

**AI 负责：**

- 竞品研究 → 诊断

- 你的 5 个选择 → 指导方针

- 创建与选择一致的 6 个月路线图 → 一致行动

**你提供：**

- 战略选择（AI 不能做这些）

- 约束和情况的上下文

- 关于哪些魔鬼代言人挑战重要的判断

**输出结构：**

```
# H1 2026 AI 产品策略

## 诊断：战略挑战
[竞争格局 + 你的约束 → 核心挑战]

## 指导方针：我们的战略方法
[你的 5 个选择综合成一致方向]
[明确权衡 - 我们不做什么]

## 一致行动：H1 2026 路线图

### Q1 2026
- 举措 1（与选择一致）
- 举措 2（与选择一致）

### Q2 2026
- 举措 3（建立在 Q1 基础上）

### 成功指标
- 指标 1：[目标]

## 关键假设
- 假设 1：[需要成立什么]

## 竞争定位
- 为什么客户选择我们 vs 替代方案
- 风险和缓解措施
```

### 5. 使用 Skills 创建高管演示文稿

使用 pptx skill 将策略文档转化为演示幻灯片：

```
Use the pptx skill to create an executive presentation from
my strategy document.

Requirements:
- 12-15 slides covering complete strategy
- Include: Title, Executive Summary, Diagnosis, Competitive
 Landscape, Strategic Direction, Tradeoffs, Roadmap (Q1/Q2),
 Success Metrics, Critical Assumptions, Why We'll Win, Risks
- Professional design for executive audience
- Save as strategy-review-slides.pptx
```

**注意：** pptx skill 需要 `python-pptx` 库。如果安装失败，可以创建幻灯片的 markdown 大纲代替。

## 真实案例

**场景：** 你是 TaskFlow 的 Gen AI PM。你发布了 AI 语音聊天用于待办事项（反响良好，使用量适中）。领导层问：我们应该如何演进 H1 2026 的 AI 策略？

**方法：**

**1. 并行研究竞争对手：**

```
Spin up 3 agents to research:
- Notion AI (positioning, features, pricing)
- Linear AI (engineering-focused features)
- Asana AI (enterprise focus, strategic direction)
```

**2. 确定诊断：** 基于竞品研究 + 约束（2 人 AI 团队，约 $50k/季度预算）：

我们无法用有限资源在广度上竞争。需要找到我们能独特获胜的防御性位置。

**3. 做出战略选择：**

- 聚焦：深入发展语音（而非广度）

- 竞争：为 SMB 差异化

- 定价：补贴以推动采用

- 范围：AI 作为增强

- 风险：稳健可防御

**4. 对每个选择进行魔鬼代言人：** 在每个选择后，听取挑战并完善或坚持。

**5. 综合策略：** AI 创建包含诊断、指导方针、一致行动的完整策略文档。

**6. 创建幻灯片：** 使用 pptx skill 生成 13 页高管演示文稿。

## 最佳实践

**应该做：**

- 基于真实数据制定诊断（竞品研究、用户反馈、市场趋势）

- 做出明确的权衡取舍 - 说出你不做什么

- 严格使用魔鬼代言人压力测试每个选择

- 确保一致行动相互强化

- 记录假设并季度回顾

**不应该做：**

- 让 AI 做战略决策（那是你的工作）

- 混淆目标与策略（"收入增长 50%"不是策略）

- 跳过艰难的权衡取舍（真正的策略需要说不）

- 忽略竞争格局或制定后不管

## 问题排查

**我的策略感觉很泛泛 - 可以适用于任何公司**

**解决方法：**

- 检查指导方针 - 它是否说了你不做什么？

- 添加具体约束："以我们 2 人团队和 $50k 预算..."

- 测试：竞争对手能否逐字复制这个？如果是，就太泛泛了。

**魔鬼代言人问题感觉不公平/过于负面**

**记住：** 从 AI 听到尖锐问题总比从 CEO 那里听到好。如果你无法捍卫你的选择，也许应该重新考虑。用挑战来完善你的思考。

**pptx skill 无法安装 / python-pptx 报错**

**解决方法：**

1. 尝试：`pip install python-pptx`

2. 如果失败：`pip3 install python-pptx`

3. 如果仍然失败，创建幻灯片的 markdown 大纲代替

**我的 5 个战略选择相互矛盾**

**解决方法：**

- 一起审视所有 5 个选择 - 它们是否讲述了一个一致的故事？

- 检查矛盾：不能既深入聚焦又到处扩展

- 测试一致性：你能否用一段话解释所有 5 个选择且说得通？
