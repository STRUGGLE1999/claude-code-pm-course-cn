# 0.0: 课程介绍

## 概述

Claude Code 是一款直接在你的电脑上操作文件和文件夹的 AI 助手。与需要手动复制粘贴上下文的传统聊天机器人不同，Claude Code 可以通过自然对话直接读取文档、分析数据并创建内容。

**核心要点：** Claude Code 将 PM 从手动执行者转变为协调者。你专注于战略决策，而 Claude 负责文档整合、数据分析和工作流自动化。

## 你将构建什么

本课程教你使用 Claude Code 构建 PM 生产力系统。

**核心能力：**

1. **文件操作** - 读取、写入和使用 @ 符号引用文件

2. **视觉分析** - 粘贴截图并分析原型图、图表、数据

3. **并行 Agent** - 同时运行多个 AI 实例进行批量处理

4. **自定义 Sub-agent** - 拥有个性化设定的永久专业团队成员

5. **项目记忆** - 通过 CLAUDE.md 持久保存产品和团队上下文

## 要求

- Claude Pro 或 Max 订阅（$20/月）

- Mac、Windows 或 Linux 电脑

- 15 分钟安装时间

**时间投入：**

- Level 1（基础）：2-3 小时

- Level 2（实践者）：4-5 小时

- Level 3（大师）：3-4 小时 [即将推出]

总计：10-12 小时成为高级用户。每周可节省 5-10 小时。

## 真实 PM 使用场景

### 战略工作

- **PRD 撰写** - 将研究、访谈和竞品分析整合成连贯的文档

- **路线图规划** - 分析市场数据和利益相关者输入

- **客户反馈分析** - 在几分钟内处理数百条反馈

- **竞品情报** - 生成 SWOT 分析和对比矩阵

### 执行与组织

- **工单生成** - 将功能规格转化为用户故事和 Jira 工单

- **多格式转换** - 将 PRD 转换为单页文档、演示文稿、发布说明

- **文件组织** - 整理研究文件、会议记录、截图

- **数据编辑** - 无需打开 Excel 即可操作电子表格和 CSV

### 沟通

- **利益相关者更新** - 起草邮件、Slack 消息、Notion 页面

- **发布说明** - 将工程变更日志转换为客户友好的语言

- **演示文稿叙事** - 从战略文档构建幻灯片

- **知识库** - 跨会话维护持久的产品知识

### 研究与分析

- **SEO 审计** - 分析营销页面和内容

- **竞品研究** - 创建对比表和功能矩阵

- **洞察整合** - 结合用户访谈、问卷、支持工单、应用评论

- **网络研究** - 自动化竞品情报收集

## Claude Code vs 传统聊天机器人

| 功能 | 传统聊天机器人 | Claude Code |
|---|---|---|
| **文件访问** | 无 | 读取/写入电脑上的任何文件 |
| **记忆** | 会话结束后消失 | 永久项目记忆（CLAUDE.md） |
| **执行** | 只提建议，你需要自己操作 | 实际执行命令并创建文件 |
| **多文件** | 单次响应 | 同时处理数十个文档 |
| **Agent** | 单实例 | 并行运行多个 Agent |
| **集成** | 有限 | 通过 MCP 连接 Notion、Linear、Slack、GitHub |
| **版本控制** | 无历史记录 | Git 集成、检查点、回退 |
| **自动化** | 手动复制粘贴 | 使用自定义命令实现真正的自动化 |

**关键区别：** 持久性和执行力。Claude Code 不仅提供建议——它与你一起完成工作。

## 真实案例：Monday.com PM

Monday.com 的一位产品经理使用 Claude Code 收集并分析了关于其产品的 34,000 条 Reddit 评论。Claude 识别出了模式、痛点和优先主题，直接指导了路线图制定。

**影响：** 数周的手动工作 → 几小时完成。这不是要取代 PM 的判断——而是处理繁重工作，让你专注于战略思考。

## 思维转变

"Claude Code 代表着开发者成为强大系统的协调者，而非命令输入者。" - Peter Steinberger，PDF Expert 创始人

这同样适用于 PM。你不是在学习编程——你是在学习协调。

**转变：**

- 从："我需要学习这个命令"

- 到："我需要达成这个结果"

Claude Code 在终端中运行，这可能让人感到陌生。但你不需要学习终端命令——你只需要进行对话。完成模块 0 后，终端只是你与 Claude 对话的地方。
