# 4.3：构建与迭代

- **完成时间：** 30-40分钟

**互动课程：** 在 Claude Code 中运行 `/start-4-3`

**本节内容：** Claude 搭建项目脚手架、构建你的应用，你通过截图进行迭代优化。

## 什么是开发服务器？

开发服务器在本地运行你的代码，让你可以像查看网站一样查看它。

关键要点：

- 它只在你的电脑上运行（不公开）
- 它让你在上线前测试和迭代
- 查看网站时它必须保持运行
- 在终端中用 `Ctrl+C` 停止它

## 什么是 Localhost？

`localhost` 意味着你自己的电脑充当服务器。

`localhost:3000` = 你机器上的 3000 端口。

这不是真正的 URL。其他人无法访问它。在 4.5 节部署时你会获得真正的 URL。

## 什么是 Plan Mode？

Plan Mode 是 Claude Code 的一个功能，Claude 在执行前先展示计划。

- 用 **Shift+Tab** 切换

- 适合你想先审查的大型任务

- 你批准计划，然后 Claude 执行

对于复杂的构建，这让你可以在编写任何代码之前提问和调整。

## 为什么迭代很重要

第一个版本永远不会完美。这是正常的，也是预期的。

专业开发始终是：构建 → 审查 → 优化 → 重复。

与 AI 的区别在于：迭代只需几秒钟而不是几天。所以不要将就——持续优化直到你满意。

## 截图技巧

**Ctrl+V**（Mac 上不是 Cmd+V！）可以直接将图片粘贴到 Claude Code。

这对迭代非常强大：

- 截取有问题的部分
- 圈出问题或指向它
- 粘贴并描述你想要什么改变

视觉反馈比用文字描述问题更快。

## 故障排除

### "npm: command not found"

Node.js 没有安装。从 [nodejs.org](https://nodejs.org/) 下载，安装后重启终端。

### "Port 3000 already in use"

另一个应用正在使用该端口。可以：

- 关闭可能正在运行服务器的其他终端窗口

- 使用不同的端口（Claude 可以帮你）

### 浏览器没有打开

手动访问 [http://localhost:3000](http://localhost:3000/)

### 空白页面

1. 检查终端是否有构建错误（红色文字）

2. 检查浏览器控制台：按 F12，点击"Console"标签，查找红色错误

### 更改没有显示

- 你刷新浏览器了吗？

- 开发服务器还在终端中运行吗？

### 样式看起来不对

用 **Cmd+Shift+R**（Mac）或 **Ctrl+Shift+R**（Windows）清除浏览器缓存，或尝试隐身窗口。
