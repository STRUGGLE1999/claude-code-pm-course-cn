# 4.2：计划

- **完成时间：** 20-25分钟

**互动课程：** 在 Claude Code 中运行 `/start-4-2`

**本节内容：** Claude 通过访谈帮你创建需求规格说明。

## 什么是需求？

需求是在构建之前对你要构建内容的描述。

把它看作是你和构建者之间的"合同"。前期越详细，后期意外越少。

这个原则适用于与以下对象协作：

- AI 工具
- 开发者
- 设计师
- 代理商

模糊的指令 = 意外的结果。具体的需求 = 你真正想要的。

## 软件中的"逻辑"是什么？

逻辑是事物行为的规则。是"如果这样，就那样"的层面。

每个应用都有逻辑：

- 如果用户点击按钮 → 显示下一个界面
- 如果答案 = A → 给性格类型 X 加分
- 如果分数最高 → 显示该结果

你用简单的中文定义逻辑。Claude 用代码实现它。

## 推荐引擎如何工作

测试题、产品推荐器和匹配系统都使用相同的模式：

1. **用户输入** → 答案、偏好、选择
2. **计分系统** → 每个输入为结果增加权重
3. **输出** → 得分最高的结果胜出

对于性格测试：

- 每个答案映射到一个性格类型
- 答案累积分数
- 最终结果 = 得分最高的性格

你不需要编写代码——只需要决定映射关系。

## 为什么可视化预览很重要

文字是有歧义的。"现代"对不同的人意味着不同的东西。

看到选项比描述偏好更快。看到的时候你就知道了。

Claude 可以生成不同视觉风格的 HTML 预览文件，并在浏览器中打开它们。你选择喜欢的——或者混合多个选项的元素。

## REQUIREMENTS.md 文件

在本节结束时，你所有的决策都会保存到一个文件：`REQUIREMENTS.md`

这包括：

- 你要构建什么
- 逻辑（如何工作）
- 视觉风格偏好
- 内容（问题、结果等）

这个文件是下一节 Claude 构建应用的规格说明。
