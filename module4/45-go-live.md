# 4.5：上线

- **完成时间：** 15分钟

**互动课程：** 在 Claude Code 中运行 `/start-4-5`

**本节内容：** 部署你的项目并获得可分享的真实 URL。

## 什么是部署？

部署是将代码从你的电脑放到互联网上。

你的代码从你的机器 → 任何人都可以访问的服务器 → 真实的 URL。

开发者经常说"部署"、"发布"或"上线"。这是你的作品变得真正可用的时刻。

## 什么是托管？

托管意味着 24/7 运行你的代码并处理访客流量的服务器。

你不需要自己管理服务器——托管服务处理所有这些。你只需要给他们代码。

## 什么是 Vercel？

Vercel 是一个网站托管服务。它：

- **免费** 用于个人项目（慷慨的免费额度）

- **由 Next.js 的创建者开发** —— 它们完美配合

- **连接到 GitHub** —— 推送变更时自动部署

把你的 GitHub 仓库给 Vercel → 它构建你的代码 → 你获得真实的 URL，如 `yourproject.vercel.app`。

## 自动部署如何工作

这就是神奇之处：

1. 你推送代码到 GitHub

2. Vercel 检测到变更

3. Vercel 自动重新构建和部署

4. 你的线上网站在约 1 分钟内更新

初始设置后无需手动操作。修改代码 → 推送 → 就上线了。

## 什么是 Vercel CLI？

Vercel CLI 是一个用于直接从终端部署的命令行工具。

Claude 使用它来部署，你无需离开 Claude Code。它通过浏览器进行认证，类似于 GitHub。

## 故障排除

### "vercel: command not found"

Vercel CLI 没有安装。运行 `npm i -g vercel`

### 认证失败

重新运行 `vercel login` 并完成浏览器授权。

### 构建失败

检查 Vercel 控制台的错误日志。通常是代码问题——告诉 Claude 错误信息，它可以帮助修复。

### "Project not found"

确保你在运行部署命令时位于正确的项目文件夹中。

### 网站显示旧版本

部署可能还在构建中。检查 Vercel 控制台的状态——通常需要约一分钟。

### 本地正常但 Vercel 上不行

环境差异。检查 Vercel 构建日志寻找线索。

## 部署后

上线后：

1. **在手机上测试** —— 同样的 URL 在任何设备上都可用

2. **分享给朋友** —— 发送链接，观察他们的反应

3. **未来更新：** 修改代码 → 推送到 GitHub → 自动部署

这就是整个流程。你现在知道如何构建和发布产品了。

## 完整循环

你已经学会了：

| 步骤 | 含义 |
|---|---|
| **计划** | 明确你要构建什么 |
| **构建** | 让 AI 创建它 |
| **迭代** | 不断完善直到满意 |
| **保存** | 备份到 GitHub |
| **上线** | 部署到互联网 |

这个循环适用于任何项目。落地页、工具、应用、原型——同样的流程。
